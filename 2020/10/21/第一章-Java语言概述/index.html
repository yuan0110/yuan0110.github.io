  1. 成员变量的修饰符

    • 默认访问修饰符：成员变量可以被同一个包（package）中的任何类访问

    • public：项目中的任何地方

    • protected：同一个package中的继承类

    • private：同一个class

    • static：类变量或者静态变量，同一个class的所有具体实例共享同一个变量。、

    • final:Java中的常量必须是类的成员，和static一起使用是不冲突的

    • transient：临时的，在默认状态下，所有的变量都是对象永久状态下的一部分；transient则指示JVM该变量不属于永久状态，主要用于实现不同对象的串行化功能

    • volatile:易变的，声明一个多线程共享变量

  3. 成员方法说明

    方法修饰符

    1. default：同一个包中的任何类
    2. public：被其他类访问
    3. protected：继承类
    4. private：只能由说明该方法的类访问
    5. static：静态方法，又叫类方法；无static修饰的叫实例方法。类方法是该类的所有对象共享的方法
    6. abstract：无方法体
    7. final：不能由子类改变
    8. synchronized：同步；实现线程同步
    9. native：本地，与本地计算机有关

    方法名应该是有意义的动词或者动词短语

5.4 接口

接口像是一种特殊的抽象类，这种类只包含常量和方法的定义，没有变量和方法的实现

  1. [public] interface interfaceName [extends SuperInterface1, SuperInterface2]
  • 有public修饰的接口，任意类都可以使用该接口
  • 缺省情况下，只有同一个包中的类可以调用该接口
  • 经过编译的生成的interfaceName.class ，即使在缺省情况下，也可以写入其他Java程序中

  1. 接口的优点

    1. 通过接口可以实现不相关类的相同行为，不用考虑不相关类的层次问题
    2. 通过接口可以指明多个类需要实现的方法
    3. 通过接口可以了解对象的交互界面，而不需了解对象所对应的类

  2. 接口的继承

    如果继承多个接口且方法需要重载时，其形参列表应该有差别

5.7 内部类和匿名类

  • 内部类的定义：一个类被嵌套到另一个类中，成为嵌套类；一般来说，嵌套类就是内部类，包含内部类的类成为外部类

  1. 内部类的特性

    1. 内部类和外部类名称不能相同
    2. 可以使用包含它的外部类的 静态成员和实例成员变量，可以使用它所在方法的局部变量
    3. 可以定义为abstract
    4. 可以声明为public和private
    5. 可以声明为static，内部类则变成了顶层类，不能再使用局部变量
    6. 若想在内部类中声明static成员（方法，变量），则内部类必须先声明为static

    静态：static修饰

  2. 抽象内部类

    内部类可以定义为抽象类型，但需要被其他内部类继承或实现

  3. 内部接口

    1. 内部类可以是一个接口，该接口可以有另一个内部类实现

  4. 局部内部类

    在方法test中，定义了一个内部类innerName，最后由这个方法返回这个内部类的对象。

    如果在用一个内部类时仅需要创建它的一个对象并返回给外部，就可以这么做。

  5. 匿名类

    匿名类是不能有名称的类，所以没有办法引用他们。必须在创建的时候，作为new语句的一部分声明

第七章 Java标准类库

7.1 简单类型的包装器类

包装器类wrappers （类都是大写的，是对象，对于自动装箱和自动拆箱的理解）

Number是Double，Float, Short, Integer , Long, Byte的父类

自动装箱和自动拆箱

  1. JDK5.0以后，基本类型的变量能够自动转换为它的包装器类的对象，称为自动装箱
  2. 自动把包装器类的对象转换成基本类型的值，被称为自动拆箱

  • 类集合只能存放对象，不能存放基本数据类型的值。当把基本类型的值放入集合时，就会发生自动装箱转换；

  • 装箱转换可能需要一个包装器类的对象，需要消耗内存，包装器类对象的值是不可变的。因此实际上不需要创建两个拥有相同值的两个不同的包装器类的对象（缓存池的好处）
