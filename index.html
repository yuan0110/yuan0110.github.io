类名的要求：规范是首字母大写，采用驼峰式写法

类的修饰符有三个：final, abstract,public(final和abstract不能修饰同一个类)

extends: 继承；只能有一个父类

implements：实现；可以试下多个接口类

类体

  1. 成员变量的修饰符

    • 默认访问修饰符：成员变量可以被同一个包（package）中的任何类访问

    • public：项目中的任何地方

    • protected：同一个package中的继承类

    • private：同一个class

    • static：类变量或者静态变量，同一个class的所有具体实例共享同一个变量。、

    • final:Java中的常量必须是类的成员，和static一起使用是不冲突的

    • transient：临时的，在默认状态下，所有的变量都是对象永久状态下的一部分；transient则指示JVM该变量不属于永久状态，主要用于实现不同对象的串行化功能

    • volatile:易变的，声明一个多线程共享变量

  2. 成员变量使用 ：类名.变量名

  3. 成员方法说明

    方法修饰符

    1. default：同一个包中的任何类
    2. public：被其他类访问
    3. protected：继承类
    4. private：只能由说明该方法的类访问
    5. static：静态方法，又叫类方法；无static修饰的叫实例方法。类方法是该类的所有对象共享的方法
    6. abstract：无方法体
    7. final：不能由子类改变
    8. synchronized：同步；实现线程同步
    9. native：本地，与本地计算机有关

    方法名应该是有意义的动词或者动词短语

  4. 实例化对象

    创建一个对象包括对象的声明，为对象分配内存空间和初赋值这三个步骤

  5. 方法重载

    方法的名字相同，但是参数不能相同

  6. 成员变量的访问权限

    -访问权限 -同一个类 -同一个包中 -不同包的子类 -不同包中的非子类

  1. 大小端，字节序列 在什么情况下使用

    • 大端模式，是指数据的高字节保存在内存的低地址中，而数据的低字节保存在内存的高地址中，这样的存储模式有点儿类似于把数据当作字符串顺序处理：地址由小向大增加，而数据从高位往低位放；这和我们的阅读习惯一致。

    • 小端模式，是指数据的高字节保存在内存的高地址中，而数据的低字节保存在内存的低地址中，这种存储模式将地址的高低和数据位权有效地结合起来，高地址部分权值高，低地址部分权值低。

满足里式替换原则：三个，可以用@override帮忙检查

为了满足里式替换原则，重写有以下三个限制：

  • 子类方法的访问权限必须大于等于父类方法；子类public>protect
  • 子类方法的返回类型必须是父类方法返回类型或为其子类型。父类List>ArrayList
  • 子类方法抛出的异常类型必须是父类抛出异常类型或为其子类型。子类>

2. 重载（Overload）

存在于同一个类中，指一个方法与已经存在的方法名称上相同，但是参数类型、个数、顺序至少有一个不同。

应该注意的是，返回值不同，其它都相同不算是重载

连接github.io出错

使用hexo遇到 报错

解决

1删除

上午点击github链接的时候，提示无法访问，原因是因为网站使用了HSTS。

解决办法比较简单，在chrome浏览器地址栏输入chrome://net-internals/#hsts，找到 delete domain security policies 项，输入域名：XXX.github.io，再点击delete。

2获取ip

通过 http://tool.chinaz.com/dns 输入xxx.github.io 获取dns

3修改hosts

C:\Windows\System32\drivers\etc 下面有个hosts文件，复制到桌面，然后再复制回C:\Windows\System32\drivers\etc，覆盖掉

完成，可以访问
